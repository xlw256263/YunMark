# 标签筛选与分页功能开发问题总结

## 问题背景

在完善云藏智能收藏夹项目时，需要实现以下功能：
1. 标签名不可重复
2. 点击标签查询书签
3. 标签和标签分类的分页管理
4. 按分类查询标签

## 遇到的问题及解决方案

### 问题1：标签名重复检查

**问题描述：** 需要确保标签名称全局唯一

**解决方案：**
- 后端在 `AdminTagService.create_tag()` 中已实现名称唯一性检查
- 创建和更新标签时都会检查是否与其他标签冲突
- 重复时返回 409 Conflict 错误

### 问题2：点击标签后页面不刷新

**问题描述：** 前端点击标签后，右侧书签列表没有更新

**根本原因：** Vue响应式数组更新问题

**解决方案：**
```typescript
// 错误做法：使用 splice 和 push（不会触发响应式更新）
selectedTagIds.value.splice(index, 1)
selectedTagIds.value.push(tagId)

// 正确做法：创建新数组替换引用
selectedTagIds.value = selectedTagIds.value.filter(id => id !== tagId)
selectedTagIds.value = [...selectedTagIds.value, tagId]
```


### 问题3：后端未正确接收标签过滤参数

**问题描述：** 前端发送 `tag_ids=[31]`，后端无法正确解析

**根本原因：** Axios默认将数组序列化为 `tag_ids[]=31` 格式，FastAPI期望 `tag_ids=31` 格式

**解决方案：** 自定义Axios参数序列化器
```typescript
function serializeParams(params: Record<string, any>): string {
  const parts: string[] = []
  for (const [key, value] of Object.entries(params)) {
    if (Array.isArray(value)) {
      for (const item of value) {
        parts.push(`${encodeURIComponent(key)}=${encodeURIComponent(item)}`)
      }
    } else {
      parts.push(`${encodeURIComponent(key)}=${encodeURIComponent(value)}`)
    }
  }
  return parts.join('&')
}

// 在Axios配置中使用
const request = axios.create({
  paramsSerializer: { serialize: serializeParams }
})
```


### 问题4：标签管理缺少分页功能

**问题描述：** 管理员标签页面没有分页，数据量大时影响性能

**解决方案：**
1. **后端：** 修改 `AdminTagService.get_all_tags_with_stats()` 支持分页参数
2. **API路由：** 添加 `page` 和 `page_size` 查询参数
3. **前端：** 在 `TagAdminView.vue` 中添加Element Plus分页组件

### 问题5：大量favicon请求影响性能

**问题描述：** 每次刷新页面都会发起大量favicon请求，很多被取消

**解决方案：**
1. **延长缓存时间：** 从7天延长到30天
2. **添加请求去重：** 使用全局Map缓存正在进行的请求
```typescript
const pendingRequests = new Map<string, Promise<string | null>>()

export async function fetchFavicon(domain: string): Promise<string | null> {
  // 如果已有请求在进行，复用该Promise
  if (pendingRequests.has(domain)) {
    return pendingRequests.get(domain)!
  }
  // 创建新请求并缓存
  const requestPromise = doFetchFavicon(domain)
  pendingRequests.set(domain, requestPromise)
  // 请求完成后清理
  requestPromise.finally(() => pendingRequests.delete(domain))
  return requestPromise
}
```


## 关键经验总结

### 1. Vue响应式系统注意事项
- 修改数组时避免使用 `splice`、`push` 等原地修改方法
- 使用 `filter`、展开运算符等创建新数组来触发响应式更新

### 2. FastAPI数组参数接收
- FastAPI的 `Query(List[int])` 期望 `?param=1&param=2` 格式
- 需要自定义前端参数序列化方式
- 添加日志调试参数类型和值

### 3. 性能优化策略
-  favicon等低频变更资源使用localStorage长期缓存
- 添加请求去重机制避免并发重复请求
- 分页处理大数据集，减少单次传输量

### 4. 调试技巧
- 前后端都添加详细的调试日志
- 使用浏览器Network面板查看实际发送的请求格式
- 检查参数类型（是否为Proxy对象）
