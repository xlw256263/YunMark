// 登录时使用（同时设置 token 和用户信息）
function setUserInfoWithToken(token: string, user: User) {
  accessToken.value = token
  userInfo.value = user
  localStorage.setItem('access_token', token)
  localStorage.setItem('user', JSON.stringify(user))
}

// 更新用户信息时使用（个人中心编辑、头像上传）
function updateUserInfo(user: User) {
  userInfo.value = user
  localStorage.setItem('user', JSON.stringify(user))
}

### 问题 2：头像上传 404 错误
**错误信息：**
```
GET http://localhost:8000/api/v1/uploads/avatars/xxx.jpg 404 Not Found
```


**原因：** 前端 `VITE_API_BASE_URL` 包含 `/api/v1`，但静态资源路径不应该有这个前缀

**解决思路：**
- 后端返回路径：`/uploads/avatars/xxx.jpg`
- 前端拼接时去掉 `/api/v1` 前缀

**解决方案代码：**
```typescript
const displayAvatar = computed(() => {
  const avatar = userStore.userInfo?.avatar
  if (!avatar) return ''
  if (avatar.startsWith('http')) return avatar
  
  const baseUrl = import.meta.env.VITE_API_BASE_URL || 'http://localhost:8000/api/v1'
  const baseWithoutApi = baseUrl.replace('/api/v1', '')
  return `${baseWithoutApi}${avatar}`
})
```

### 问题 6：注册时邮箱/用户名唯一性校验范围
**需求：** 注销账号后，相同邮箱/用户名应该允许重新注册

**问题：** 数据库层面的唯一索引约束阻止了插入

**解决思路：** 注销时修改用户名和邮箱，解除唯一性约束

**解决方案代码：**
```python
# back/app/services/user_service.py

# 1. 查询时只检查激活用户
def get_user_by_email(db: Session, email: str) -> User:
    return db.query(User).filter(
        User.email == email, 
        User.is_active == 1  # 只查激活用户
    ).first()

# 2. 注销时修改用户名和邮箱
def delete_user(db: Session, user: User) -> bool:
    user.is_active = 0
    timestamp = int(time.time())
    user.username = f"{user.username}_deleted_{timestamp}"
    user.email = f"{user.email}_deleted_{timestamp}"
    db.commit()
    return True
```

## 🎯 关键技术点总结

### 1. **头像 URL 处理**
- 后端返回相对路径：`/uploads/avatars/xxx.jpg`
- 前端需去掉 `/api/v1` 前缀：`baseUrl.replace('/api/v1', '')`
- 支持完整 URL 和相对路径两种格式

### 2. **Store 函数职责划分**
- `setUserInfoWithToken(token, user)` - 登录时使用
- `updateUserInfo(user)` - 更新信息时使用
- 避免函数名冲突导致状态管理混乱

### 3. **软删除与唯一性约束**
- 数据库唯一索引不受 `is_active` 影响
- 注销时修改用户名/邮箱为唯一值（加时间戳）
- 查询时过滤 `is_active=1` 的用户

### 4. **FormData 上传**
```typescript
const formData = new FormData()
formData.append('file', file)
headers: { 'Content-Type': 'multipart/form-data' }
```


### 5. **Vite 热更新问题**
- 修改后硬刷新浏览器：`Ctrl + Shift + R`
- 必要时清除缓存：`rm -rf node_modules/.vite`
