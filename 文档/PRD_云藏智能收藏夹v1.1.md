## 1. 项目背景与目标
### 1.1 产品定位
“云藏”是一款**个人网址收藏与社区分享工具**。用户可收藏网址、打标签、建分类，也可选择将收藏公开分享给社区。管理员审核后可将优质内容推送到官方精选页。

### 1.2 核心目标
+ **收藏管理**：高效的个人书签增删改查，支持分类与标签双维度组织。
+ **社区分享**：用户可公开收藏，社区成员可浏览、点赞、收藏他人的分享。
+ **数据洞察**：可视化点击统计与收藏习惯分析。
+ **合规管理**：管理员审核公开内容，维护黑名单，保障平台合规。

**核心规则**：

1. **标签（Tag）**
    - 由管理员统一创建和维护
    - 标签库全局共享，所有用户使用同一套标签体系
    - 用户在创建/编辑书签时，只能从管理员定义的标签库中选择
    - 常见标签示例：AI、生活、影视、编程、效率工具、说明文档、技术、设计等
    - 一个书签可以关联多个标签
2. **分类（Category）**
    - 由用户个人创建和管理
    - 分类属于用户私有，不同用户的分类互不影响
    - 分类用于用户个人的书签组织管理
    - 一个书签只能归属一个分类（可选，也可以不分类）
3. **书签（Bookmark）**
    - 由用户创建
    - 归属关系：1 个书签 → 1 个分类（可选）
    - 标签关系：1 个书签 → N 个标签（从管理员标签库选择）
    - 书签包含：URL、标题、描述、图标、点击次数、分类、标签列表

### 1.4 技术约束
+ 后端已实现用户注册登录（JWT）、书签 CRUD、分类 CRUD、标签 CRUD、点击计数。
+ **当前后端未实现的功能**（需后续迭代）：管理员角色、标签管理接口、公开分享/点赞、审核流、统计聚合接口、LLM 自动标签推荐。前端设计需预留接口占位。

**特殊功能**：

+ **点击跳转**：点击书签卡片主体区域，在新标签页打开 URL（`target="_blank"`），同时调用 `PATCH /bookmarks/{id}/click` 增加点击次数。
+ **编辑/删除按钮**：在卡片右上角显示更醒目的操作按钮，hover 时高亮显示。
+ **拖拽排序**：使用 `vuedraggable` 实现，排序结果暂存 `localStorage`，待后端提供 `sort_order` 字段后同步。
+ **批量删除**：多选模式勾选后，逐条调用 `DELETE /bookmarks/{id}`，带二次确认。
+ **导入 HTML 书签**：前端解析 `<A HREF="...">标题</A>` 格式，批量提交（需后端提供批量导入接口，暂标记为 TODO）。

```typescript
// 伪代码
router.beforeEach((to) => {
  const authRoutes = ['/my', '/stats', '/profile']
  const adminRoutes = ['/admin']

  if (authRoutes.some(r => to.path.startsWith(r)) && !isAuthenticated()) {
    return { path: '/', query: { login: '1' } }
  }
  if (adminRoutes.some(r => to.path.startsWith(r)) && !isAdmin()) {
    return { path: '/403' }
  }
})
```

### 8.3 API 交互封装
```typescript
// request.ts 核心逻辑
const request = axios.create({ baseURL: '/api/v1', timeout: 10000 })

// 请求拦截：注入 JWT
request.interceptors.request.use((config) => {
  const token = localStorage.getItem('access_token')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// 响应拦截：统一错误处理
request.interceptors.response.use(
  (res) => res.data,
  (err) => {
    if (err.response?.status === 401) {
      localStorage.removeItem('access_token')
      router.push({ path: '/', query: { login: '1' } })
    }
    ElMessage.error(err.response?.data?.detail || '网络异常，请稍后重试')
    return Promise.reject(err)
  }
)
```

### 8.4 错误处理
| 场景 | 处理方式 |
| --- | --- |
| 请求成功 | `ElMessage.success()` |
| 业务错误（400） | `ElMessage.error(detail)` |
| 未登录（401） | 自动跳转登录 |
| 无权限（403） | 跳转 403 页 |
| 未找到（404） | 跳转 404 页 |
| 网络异常 | `ElMessage.error('网络连接失败')` |
| 空状态 | 展示空状态插画 + 引导按钮 |
| 加载中 | 骨架屏（`el-skeleton`） |

| 4 | 管理员删除已使用标签 | 提示“该标签已被使用，无法删除” |
| 5 | 管理员审核通过一条分享 | 该条从待审核列表移除 |
| 6 | 管理员驳回并填写原因 | 分享状态变更为“已驳回”，用户可见驳回原因 |
| 7 | 批量勾选 3 条并通过 | 3 条同时从列表消失 |
